# Плагин поддержки Искусственных Индусов (Генераторов Псевдослучайных Текстов и подобных)

## Установка

Данный плагин не включён в docker-сборку и не совместим с ней, так что для его использования нужно запускать приложение
из исходного кода.
Основной проект и этот плагин могут предпочитать разные версии некоторых транзитивных зависимостей, та что лучше
переставить все зависимости следующей командой (в корне репозитория):

```shell
pip install -r requirements.txt irene_plugin_llm/requirements.txt
```

По-умолчанию, плагин настроен на использование модели `qwen2.5:14b` на локальном сервере ollama.
Если сервер запущен на другой машине или нужно использовать другую модель - измените настройки плагина `llm_ollama`.
Если нужно использовать другой сервис для запуска LLM, выберите соответствующий адаптер в настройках плагина
`llm_fallback_context` и, при необходимости, добавьте дополнительные параметры там же или в настройках соответствующего
плагина.

Обратите внимание, что не все модели поддерживают вызов инструментов.

## Поддерживаемые сервисы моделей

На данный момент реализована поддержка подключения к следующим сервисам:

| Сервис                 | Плагин                               | Пример настройки                             | Примечания       |
|------------------------|--------------------------------------|----------------------------------------------|------------------|
| Ollama                 | [llm_ollama](./plugin_llm_ollama.py) | `{"type": "ollama", "model": "qwen2.5:14b"}` |                  |
| OpenAI (и совместимые) | [llm_openai](./plugin_llm_openai.py) | `{"type": "openai", "model": "gpt-4o-mini"}` | Не протестирован |

В настройках плагина `llm_fallback_context` можно указать только поле `type` - все не указанные параметры будут взяты
из настроек плагина LLM.

Поддержку других LLM из [каталога поддерживаемых](https://python.langchain.com/docs/integrations/chat/) langchain не
сложно -
достаточно написать плагин, по аналогии с имеющимися плагинами LLM.

## Добавление инструментов (скиллов) для LLM

Добавить инструменты можно плагином, реализующим операцию `lc_tools`, каждый шаг операции должен быть инструментом
в формате
langchain ([BaseTool](https://python.langchain.com/api_reference/core/tools/langchain_core.tools.base.BaseTool.html)):

```python
from langchain_core.runnables import RunnableConfig
from langchain_core.tools import tool
from eva.plugin_loader.magic_plugin import operation
from eva import VAApiExt

name = 'my_ai_math_plugin'
version = '0.0.0'


@operation('lc_tools')
@tool(parse_docstring=True)
def add_numbers(a: int, b: int) -> int:
    """
    Складывает два числа и возвращает результат сложения.
    
    Args:
        a: первое число
        b: второе число
    """
    # ^ langchain автомагически извлекает описание инструмента из docstring'а
    # (если установлен параметр parse_docstring, как выше), так что в docstring стоит добавлять всё то и только то,
    # что LLM'ке стоит знать об инструменте и его параметрах.
    return a + b


@operation('lc_tools')
@tool(parse_docstring=True)
def do_something_else(
        arg1: str,
        run_config: RunnableConfig
) -> str:
    """
    Делает что-то другое.
    
    Args:
        arg1: аргумент, который передаст LLM, его описание тут важно
        run_config: этот аргумент будет автоматически передан langchain, LLM о нём ничего не узнает
    """
    # Из RunnableConfig можно получить доступ к VAApiExt, через который можно взаимодействовать
    # с пользователем в обход LLM:
    # - воспроизводить звуки и сообщения
    # - запускать отложенные взаимодействия
    va: VAApiExt = run_config['configurable']['eva_va_api']
    return "blablabla"
```

Получить список доступных инструментов с их описаниями можно при помощи запроса
на [/api/llm_fallback_context/tools](http://localhost:8086/api/llm_fallback_context/tools).
Это позволяет убедиться, что описания инструментов и параметров распарсились корректно.
